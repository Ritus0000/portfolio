# Confidentiality Policy

This portfolio can include work inspired by real projects, but it must not expose private material.

## Do Not Publish

- internal laboratory documentation
- private project files
- unpublished datasets
- confidential screenshots
- original internal configurations
- copied private code
- exact project parameters that were not cleared for public use
- raw work reports unless explicitly approved

## Safe Public Material

- self-created demo scenes
- simplified reproductions of workflow mechanics
- high-level diagrams
- public assets with proper licensing
- original explanatory text written specifically for the portfolio
- approved sanitized visuals
- architecture or design work that is already public or safe to show

## Recommended Wording

Use this wording for work that cannot be shown directly:

> This public case study is a simplified, non-confidential reproduction of the workflow mechanics. It uses only public or self-created demo materials and does not contain internal data, code, documentation, or project-specific parameters.

## Practical Rule

If the file came from private work, do not publish it.

If the file was created specifically as a public demonstration, it can be considered for the portfolio after review.
