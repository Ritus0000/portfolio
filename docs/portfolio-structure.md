# Portfolio Structure

The portfolio should stay broad, but not chaotic.

## Main Narrative

The central identity:

> AI, 3D simulation, visual systems, and spatial design.

This allows the portfolio to include synthetic data, rendering, architecture, game-adjacent visuals, scientific AI, and product systems without looking like unrelated fragments.

## Recommended Order

1. Synthetic inspection / computer-vision data pipeline
2. 3D rendering and materials
3. Architecture and spatial design
4. AI systems and automation
5. Scientific / biomedical AI

## Visual Rule

Each case study should include:

- one strong hero visual
- a short problem statement
- what I built
- workflow / pipeline
- outputs
- tools
- what this proves

## Avoid

- dumping every project equally
- mixing confidential work with public demos
- presenting unfinished ideas as finished products
- overclaiming tools that were only adjacent or theoretical
